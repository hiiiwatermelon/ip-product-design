# 产品故事引擎参考

本参考用于把心理机制转化为产品设计决定。理论只能提供设计启发，不能替代用户证据、可用性测试或真实传播数据。

## 一、故事不是文案层，而是因果结构

产品故事至少包含：

```text
主角 + 具体时刻 + 张力 + 参与 + 证据 + 转折 + 行动
```

- **主角**：用户本人，而不是IP或产品；
- **具体时刻**：必须做判断、选择或行动的时刻；
- **张力**：不知道、做不到、选不定、看不清或担心后果；
- **参与**：用户选择、输入、模拟、比较或生成；
- **证据**：转折成立的依据；
- **转折**：改变用户原来的解释、判断或行动；
- **行动**：用户恢复可控感并知道下一步。

## 二、六种心理机制选择器

| 机制 | 适合解决 | 产品实现 | 不要误用 |
|---|---|---|---|
| 场景代入 | 用户觉得“与我无关” | 首屏进入具体决定时刻，使用第二人称和真实约束 | 不等于编造戏剧化故事 |
| 自我相关 | 结果过于通用 | 让用户提供会改变结果的经历、选择或事实 | 不以个性化为名过度采集隐私 |
| 信息缺口 | 用户没有继续动力 | 明确完成后能回答的问题，暂不泄露结论 | 不做虚假悬念或标题党 |
| 低压力微行动 | 首步阻力太高 | 第一题简单、可跳过、低隐私，先给局部价值 | 不隐藏真实成本，不诱导承诺 |
| 同类比较 | 用户缺少坐标 | 给出可比人群、阶段或风险区间，并说明依据 | 不做虚假排名、羞辱性标签或错误对比 |
| 风险与可控感 | 用户看到风险后逃避 | 风险、可控制部分、方法和下一步同时出现 | 不把恐惧用于所有产品，不夸大危害 |

选择原则：只使用能解决当前体验问题的机制。一个机制必须对应一个可观察的页面或交互决定。

## 三、四种叙事原型

### 1. 诊断型

```text
原来没意识到 → 看见迹象 → 重构原因 → 获得方案
```

适合风险诊断、职业诊断、企业诊断。关键不是把问题说严重，而是让用户看见原因与可控部分。

### 2. 模拟型

```text
面临决定 → 做出选择 → 看见后果 → 重新演练
```

适合谈薪、咨询、销售、沟通和决策模拟。用户的选择必须真实影响反馈。

### 3. 坐标型

```text
不知道位置 → 提供事实 → 获得坐标 → 明确方向
```

适合阶段、能力、类型和行业对照。比较对象必须可比，结果需说明来源、边界和不确定性。

### 4. 生成型

```text
信息混乱 → 回答关键问题 → 方法介入 → 得到可用成果
```

适合方案、脚本、模板和行动计划。故事转折来自“我终于把问题说清或做成了”，不是强行制造惊吓。

## 四、四类张力

不要把故事感等同于负面情绪。根据真实场景选择：

1. **不确定张力**：我不知道自己该怎么判断；
2. **选择张力**：两个选项各有代价；
3. **身份张力**：我以为自己是A，但行为显示可能是B；
4. **机会张力**：我可能错过一个本可以抓住的结果。

风险型产品可以使用威胁张力，但必须同时提供有效、可执行的控制方式。

## 五、认知转折的五种来源

可信转折只能来自：

1. 用户自己的选择前后不一致；
2. 用户输入与可靠基准之间的差异；
3. 多个事实之间原先未看见的关系；
4. 模拟行为造成的可解释后果；
5. IP独有且能说明依据的方法论判断。

“AI说你属于某种人”不是证据。“你一定会怎样”不是可信转折。

## 六、可携带故事对象

一个传播对象至少应回答：

- 发生在谁身上；
- 原来怎样；
- 发现了什么；
- 为什么可信；
- 接下来做什么。

推荐一句话结构：

```text
我原来以为____；在____之后，我发现____；因为____；接下来我准备____。
```

公开版应允许隐去金额、公司、健康、家庭、身份和其他敏感字段。用户不分享，也必须能获得完整产品价值。

## 七、传播力启发式

可用以下关系检查设计，不作为科学公式：

```text
传播潜力 ≈ 自我相关性 × 转折清晰度 × 证据可信度 × 表达压缩度 × 身份安全感
```

任何一项接近零，都可能让传播失效：

- 不关于我：没有分享动机；
- 没有转折：没有故事；
- 不可信：不愿背书；
- 说不清：无法转述；
- 不安全：不愿公开。
