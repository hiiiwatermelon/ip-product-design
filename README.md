# 知识IP内容型产品设计框架

> 基于 Nir Eyal《上瘾》模型的适配框架，用于知识IP为获客而设计的产品——以获客为核心战略目的、以内容特质为设计标准的低频高价值产品。

## 这是什么

知识IP（职场博主、咨询师、教练等）做产品时，常被"DAU/留存/习惯路径"这套高频产品的指标体系误导——低频高价值的产品（如谈薪模拟器）不可能让用户每天打开，强行套用只会得出"失败"的结论。

本框架对上瘾模型做了适配性改造：

- **优化目标**：从"频率"改为"IP信任势能 + 单次成交价值"
- **评估方式**：从"习惯测试"改为"循环健康度评估"
- **核心结构**：三层产品矩阵解耦频率与价值
- **品类定义**：引入"内容型产品"——同时满足工具标准和内容标准的产品

**内容型产品的三大特质**：

1. **可传播性** — 产品结果自带叙事结构，不是干数据
2. **人格表达力** — 交互有IP指纹，用户在和"IP的方法论化身"交互
3. **叙事感** — 体验有情绪弧线，不是纯功能流程

## 框架包含

- 品类定义与区分（内容型产品 vs 内容产品 vs 工具产品）
- 标准上瘾模型的适配边界（适用/不适用）
- IP驱动的低频高价值循环 + 三层产品矩阵
- 设计五步法（选锚点 → 定频率 → 行动入口 → 投入阶梯 → 闭环出口）
- 三大特质 × 上瘾模型映射
- 赋能IP的五条通道
- 替代评估指标（取代标准习惯测试）
- 三种反面模式 + 道德审视

## 安装方法

将整个 `ip-product-design` 目录复制到你的 WorkBuddy skills 目录下：

```bash
cp -r ip-product-design ~/.workbuddy/skills/
```

安装后，在 WorkBuddy 对话中提到"产品设计""IP做产品""内容型产品"等话题时，skill 会自动加载框架内容到对话上下文。

也可以手动调用：

```
/ip-product-design
```

## 与 eyal-hooked skill 配合使用

如果同时安装了 [eyal-hooked skill](https://github.com/your-username/eyal-hooked)，讨论原著上瘾理论时先加载 eyal-hooked，再用本框架做适配映射。
